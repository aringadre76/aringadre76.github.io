## README

# Deskscribe

An interactive card-driven web novel where players draw cards that shape narrative constraints. Uses LangChain to generate dynamic story content with persistent character and lore tracking via vector embeddings.

## Overview

Deskscribe combines card game mechanics with AI-powered storytelling to create unique, player-driven narratives. Each card drawn maps to specific narrative constraints that the LLM weaves into the story while maintaining continuity with past events and character development.

## Features

For detailed feature documentation, see [FEATURES.md](FEATURES.md).

### Current Features
- Standard 52-card deck with narrative mappings
- Custom "Fate" deck with 25 thematic cards
- Draw 1-3 cards per turn
- LangChain-powered story generation
- Vector database for scene retrieval and continuity
- Player choices that influence the narrative
- Export story to Markdown
- Beautiful pastel-themed UI

### Future Phases
- **Phase 2**: Branching narratives, per-character memory, custom deck editor
- **Phase 3**: Image generation for scenes, multiplayer, community decks
- **Phase 4**: Marketplace, webnovel imports, fine-tuned LLMs

## Installation

### Prerequisites
- Python 3.10 or higher
- Node.js 18 or higher
- A local LLM endpoint (optional for Phase 1, uses stub responses)

### Quick Start

```bash
# Install system dependencies (if needed)
sudo apt install python3-pip python3-venv

# Run the application
cd /home/robot/deskscribe
./scripts/start_all.sh
```

Then open http://localhost:3000 in your browser!

### Manual Setup

#### Backend
```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Backend API: `http://localhost:8000`  
API Docs: `http://localhost:8000/docs`

#### Frontend
```bash
cd frontend
npm install
npm run dev
```

Frontend: `http://localhost:3000`

## Usage

1. **Start a New Game**: Enter your protagonist's name and optional world seed
2. **Choose Deck Type**: Select Standard (playing cards) or Fate (thematic cards)
3. **Draw Cards**: Click to draw 1-3 cards that will shape the next scene
4. **Make a Choice**: Select from available story choices
5. **Generate Scene**: Click to generate the next scene based on cards and choice
6. **Export**: Save your entire story as Markdown at any time

## API Endpoints

### `POST /api/new-game`
Start a new game session.

**Request:**
```json
{
  "protagonist_name": "Arin",
  "world_seed": "A city of eternal twilight"
}
```

### `POST /api/draw`
Draw random cards from the deck.

**Request:**
```json
{
  "count": 2,
  "deck_type": "standard"
}
```

### `POST /api/generate`
Generate a new story scene.

**Request:**
```json
{
  "player_id": "uuid",
  "cards": ["A♠", "Q♥"],
  "player_action": "Follow the shadow to the docks",
  "deck_type": "standard"
}
```

### `GET /api/history?player_id={uuid}`
Retrieve all scenes for a player.

### `GET /api/player/{player_id}`
Get player state and metadata.

## Card Mappings

### Standard Deck
- **Spades**: Dark themes (Betrayal, Danger, Death, Mystery)
- **Hearts**: Positive themes (Ally, Romance, Healing, Hope)
- **Diamonds**: Material themes (Wealth, Discovery, Power, Success)
- **Clubs**: Action themes (Combat, Challenge, Growth, Strategy)

## Local LLM Integration

The system is designed to work with a local LLM running on your GPU. To connect:

1. Set up your LLM inference server (e.g., text-generation-webui, LM Studio, or vLLM)
2. Create a `.env` file in the `backend/` directory:

```env
LLM_BASE_URL=http://your-gpu-pc-ip:8000/v1
LLM_MODEL_NAME=your-model-name
LLM_TEMPERATURE=0.7
```

3. Update `langchain_pipeline.py` to use the LLM client (currently uses stub responses)

### Recommended Models
- Llama 3 70B or 8B
- Mistral 7B v0.3
- Nous Hermes 2
- Any model with strong creative writing capabilities

## Development

### Running Tests
Tests and evaluation framework will be added in a future update.

### Database
SQLite is used by default for development. The database file is created as `deskscribe.db` in the backend directory.

Vector embeddings are stored in `./chroma_db/` using ChromaDB.

### Customizing Card Decks
Edit `backend/app/cards_map.json` or `backend/app/fate_deck.json` to customize card mappings.

Each card should have:
```json
{
  "Card Name": {
    "label": "Short Label",
    "constraint": "Detailed narrative constraint for the LLM"
  }
}
```
